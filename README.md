# LLM-Powered Prompt Router

A production-style Python service that classifies incoming requests, routes them to a specialized AI persona, and logs every decision to `route_log.jsonl`.

## Features

- Two-step intent routing: classify first, then generate with a focused expert prompt.
- Four expert personas: `code`, `data`, `writing`, and `career`.
- Safe fallback to `unclear` when the classifier returns malformed JSON or low confidence.
- Manual override with prefixes like `@code Fix this bug`.
- FastAPI endpoint plus CLI entrypoint.
- Chat-style frontend served at `/` for live demos.
- JSON Lines logging for every routed request.
- Docker, Docker Compose, unit tests, and a committed sample log file.

## Design

The router follows a simple production pattern:

1. `classify_intent(message)` sends the user message to a lightweight model with a strict JSON-only classifier prompt.
2. The raw text response is parsed defensively. If parsing fails or the label is invalid, the router defaults to `{"intent":"unclear","confidence":0.0}`.
3. `route_and_respond(message, intent)` either asks a clarification question for `unclear` requests or selects a specialist system prompt and makes a second LLM call for the final answer.
4. Every request is appended to `route_log.jsonl` as one JSON object per line.

The app defaults to the OpenAI Responses API using:

- `gpt-4.1-nano` for classification
- `gpt-4.1-mini` for generation

Both are configurable through environment variables.

## Setup

### Local

1. Create a virtual environment.
2. Install dependencies.
3. Copy `.env.example` to `.env` and add your OpenAI API key.
4. Start the API or use the CLI.

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
uvicorn app.main:app --reload
```

### API Usage

Run the server:

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Send a request:

```bash
curl -X POST http://localhost:8000/route \
  -H "Content-Type: application/json" \
  -d "{\"message\":\"how do i sort a list of objects in python?\"}"
```

Example response shape:

```json
{
  "user_message": "how do i sort a list of objects in python?",
  "normalized_message": "how do i sort a list of objects in python?",
  "intent": "code",
  "confidence": 0.94,
  "routed_intent": "code",
  "final_response": "...",
  "manual_override": false,
  "timestamp": "2026-03-13T00:00:00+00:00"
}
```

### Chat UI

Start the server and open:

```text
http://127.0.0.1:8000/
```

The homepage provides a chat-style interface that posts to `/route`, displays the detected intent and confidence, and keeps a running conversation history for demos.

### CLI Usage

```bash
python cli.py "My boss says my writing is too verbose."
python cli.py --json "@code fxi thsi bug pls: for i in range(10) print(i)"
```

## Docker

Build and run with Docker Compose:

```bash
copy .env.example .env
docker compose up --build
```

The service will be available at `http://localhost:8000`.

## Environment Variables

Documented in `.env.example`:

- `OPENAI_API_KEY`: required for live LLM calls.
- `GROQ_API_KEY`: optional alternative to `OPENAI_API_KEY` when using Groq's OpenAI-compatible API.
- `OPENAI_BASE_URL`: optional override for compatible gateways.
- `CLASSIFIER_MODEL`: low-cost model for intent classification.
- `GENERATION_MODEL`: model for persona responses.
- `CONFIDENCE_THRESHOLD`: if confidence is below this value, route as `unclear`.
- `ROUTE_LOG_PATH`: JSONL log output path.
- `APP_HOST` and `APP_PORT`: server defaults for local runs.

## Testing

Run the unit tests:

```bash
pytest
```

The test suite covers:

- Valid JSON parsing
- Malformed JSON fallback
- Clarification behavior for unclear requests
- Logging format
- Manual override behavior
- Confidence threshold routing
- A 15-message submission-style routing matrix

## Sample Log File

`route_log.jsonl` is a committed sample generated by:

```bash
python scripts/generate_sample_log.py
```

This uses deterministic demo outputs so the repository includes a real log artifact even without an API key.

## Notes

- The writing prompt honors the assignment by coaching rather than defaulting to full rewrites.
- The career prompt asks clarifying questions when context is missing.
- The classifier parser strips markdown fences and safely handles malformed responses.
- If a request mixes multiple intents or is too vague, the router asks for clarification instead of guessing.
- Groq can be used by setting `GROQ_API_KEY`, `OPENAI_BASE_URL=https://api.groq.com/openai/v1`, and Groq-supported model names.
